Overview

POPO는 반려동물과 함께하는 액티비티를 앱과 채널을 오가지 않고 탐색하고 예약할 수 있도록 기획한 모바일 서비스다. 여러 플랫폼에 흩어져 있던 정보 탐색 과정을 하나의 경험으로 연결해, 사용자가 원하는 액티비티를 빠르게 발견하고 예약까지 이어갈 수 있는 흐름을 설계했다. 프로젝트 전반에서 사용자 리서치와 문제 정의를 바탕으로 UX 전략을 수립하고, 실제 동작하는 서비스와 다양한 예외 시나리오를 포함한 프로토타입을 구현했으며, 사용자 테스트를 통해 아이디어를 검증하고 실제 서비스 관점에서 경험을 개선했다.

03 · User Research

설문으로 가설을 확인하다

도출한 가설을 바탕으로, 정보 탐색 과정에서 사용자가 중요하게 고려하는 기준과 어떤 기준으로 선택하는지 확인하기 위해 설문조사를 진행했다.

Q. 반려견과 함께할 장소·활동 정보를 탐색할 때 느끼는 어려움은?

04 · Solution

파편화된 탐색부터 예약까지, 하나의 가이드로

파편화된 정보 탐색부터 패키지 예약까지, 반려견 성향에 최적화된 통합 액티비티 가이드를 제안한다.

07 · Design & Build

목표 흐름을 따라가 본 핵심 화면

10개의 화면 중, 서비스의 문제 해결 논리를 가장 잘 보여주는 흐름 위주로 정리했다.

🔍 뭘 검색할지 몰라도, 태그가 먼저 제안해요
STEP 1 · 검색

해시태그로 찾는, 반려동물 특화 검색

일반 지도·검색 서비스는 "카페", "공원" 같은 일반 키워드 중심이라 반려동물 동반 여부까지 다시 확인해야 했다. POPO는 #독스포츠, #댕마라톤 같은 반려동물 활동 태그와 인기 검색어를 검색 진입점에 바로 노출해, 첫 화면에서부터 목적에 맞는 탐색이 시작되도록 설계했다.

Why — 사용자가 "무엇을 검색해야 할지" 몰라 이탈하는 지점을 줄이기 위해, 자유 검색어 대신 태그·인기 검색어를 먼저 제안하는 구조를 우선순위로 뒀다.
🐾 성향까지 맞춰 추천해요
STEP 2 · 탐색

성향과 위치, 두 축으로 좁히는 탐색

모든 반려견이 같은 곳을 좋아하지 않는다는 점에서 출발해, "친구가 좋아요 · 예민해요 · 소심해요 · 활동이 많아요"처럼 반려견 성향 기반 필터를 만들었다. 지도 화면에서는 카페 · 실내 · 대형견 가능 · 액티비티 필터로 리스트와 지도를 동시에 좁힐 수 있게 했다.

  • 성향 태그 필터 — 견종·성격에 맞는 장소로 좁히기
  • 지도 + 리스트 동시 뷰 — 위치 감각과 정보 비교를 동시에
  • 대형견 가능 여부 등 반려동물 특화 필터 노출
👀 28명이 보고있어요
⭐ 만족도 96%
STEP 3 · 상세 확인

예약 전 불안을 없애는 상세 정보

반려동물 동반 서비스는 "우리 아이가 괜찮을까"라는 불안이 예약 직전 이탈로 이어지기 쉽다. 그래서 상세 화면에 실시간 조회수("28명이 보고있어요"), 만족도 배지(96%), 담당 트레이너 프로필과 경력, 실제 이용 후기와 별점, 취소 정책·준비물·주의사항을 한 화면 안에서 모두 확인할 수 있도록 설계했다.

Why — 신뢰 요소(사회적 증거, 전문성, 정책 투명성)를 예약 버튼 이전에 배치해, "확인하러 다른 곳으로 이탈"하는 경로 자체를 없앴다.
1/4 → 4/4 진행률 항상 표시
STEP 4-5 · 예약 · 결제

4단계로 명확하게 끊어낸 예약·결제

반려견 선택 → 날짜·시간 선택 → 결제 방식 → 최종 확인, 총 4단계로 진행률을 항상 화면 상단에 노출했다. 결제 화면에서는 POPO Pay·카카오페이·네이버페이·토스페이 같은 실사용 빈도가 높은 간편결제를 우선 배치하고, 상품 금액·예약 수수료·할인 혜택을 항목별로 분리해 투명하게 보여줬다.

Why — 결제 직전 이탈은 대부분 "이 금액이 맞는지 확신이 안 서서" 발생한다. 수수료를 0원으로 명시하고 혜택을 항목으로 분리해, 결제 버튼을 누르기 전 재확인 없이 바로 진행할 수 있게 했다.
📅 예약 · 플랜 · 리뷰 한 곳에서
STEP 6 · 일정 확인

예약 이후의 경험까지 설계

결제로 끝나는 것이 아니라, "일정" 탭에서 예약·플랜·리뷰를 한 곳에서 관리하도록 했다. 예약을 취소할 때는 환불 정책(3일 전 전액, 1일 전 50%, 당일 불가)을 다시 한 번 안내하는 확인 모달을 배치해 정책에 대한 오해로 인한 불만을 예방했다. 가이드 화면에는 방문 전 준비물, 이용 시 주의사항, FAQ를 정리해 예약 전후 어디서든 궁금증을 해소할 수 있게 했다.

08 · Implementation

디자인에서 끝내지 않았다

화면을 그리는 것에서 멈추지 않고, 실제로 동작하는 서비스로 구현하고 배포했다.

🛠 Front-end Implementation

Figma에서 설계한 화면을 실제 마크업·스타일·인터랙션으로 직접 구현했다. 탭 전환, 필터 적용, 4단계 예약 플로우, 결제 수단 선택 같은 핵심 동선이 실제로 클릭하고 이동 가능한 살아있는 프로토타입으로 존재한다.

🚀 Deploy

Vercel에 배포해 실제 URL로 언제든 접근하고 테스트할 수 있는 상태로 만들었다. 디자인 파일이 아니라 링크 하나로 서비스를 경험할 수 있다는 것이 이 프로젝트의 핵심 차별점이다.

📊 Analytics

Google Tag Manager를 연동해 실제 사용자 행동(검색, 필터 적용, 예약 시작, 결제 완료 등)을 데이터로 수집할 수 있는 환경을 구축했다. 감으로 판단하지 않고, 데이터로 검증하는 구조를 만드는 것이 목표였다.

09 · Reflection

POPO는 예쁜 화면을 만드는 프로젝트가 아니라, 문제를 정의하고 근거를 세우고 실제로 만들어 검증하는 사이클을 처음부터 끝까지 경험해보고 싶어서 시작한 프로젝트다. 디자인은 결정을 내리는 과정의 일부일 뿐이고, 그 결정이 실제로 맞았는지는 구현하고 배포하고 데이터를 봐야만 알 수 있다는 것을 이 프로젝트를 통해 다시 확인했다.
